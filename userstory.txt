**User Story #1** As a user, I want to log in to the system so that I can access the protected functionality.

**Acceptance Criteria:**

- The system displays a login form with username and password fields
- The system validates credentials against stored values in login.properties
- The system redirects to menu page upon successful login
- The system displays an error message for invalid credentials
- The system maintains the user session after successful login

**Business Rules:**

- Login credentials are stored in login.properties file

**Input Model:**

| Field Name | Type | Required | Description |
| --- | --- | --- | --- |
| username | String | Yes | User's login name |
| password | String | Yes | User's password |

---

**User Story #2** As a user, I want to log out of the system so that I can securely end my session.

**Acceptance Criteria:**

- The system provides a logout button on all pages
- The system invalidates the user session upon logout
- The system redirects to the login page after logout
